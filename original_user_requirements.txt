# AutoDesk

AI-powered development platform that replaces human developers with autonomous agent teams.

AutoDesk is a desktop application made using `Electrobun` (you can use `electrobun` skill for this) where AI agents handle the entire software development lifecycle — from planning and coding to reviewing and testing. Each project gets its own team of AI agents that work autonomously based on human instructions. Human in loop for plan approvals and deployments though deployments can also be run by `devops-engineer` agent especially designed to run in deployment page.

## Tech

- Electrobun for cross platform app
- Vercel AI SDK for unified ai interfernce, etc.
- TypeScript and type checking, linting
- Frontend framework (claude to decide but it must be modern, professional, famous and beautiful, we need Light theme only)
- SQLite with WAL performant mode and corruption-safe, persistent conection or whatever best practices are.
- Claude to decide further.
- Modular Code
- Tests for all important features
- Reusable components
- KISS, DRY, SOLID principles

## Rough core Ideas of App:

- 99% work done by agents - humans only approve/deploy/communicate
- Agents follow specific workflows defined in workflows.
- Humans can communicate with agents via various channels
- Heartbeats (similar to openclaw but do we need this based on autodesk project requiremtns? claude to decide)
- Consitution (set of hard boundaries/guardrails agents can not bypass) defined in global settings. claude to decide with it should be defaulted with.
- Agents fully customizable for their behaviur/options
- Realtime activity of counts, etc. Also let's say human asked project manager to perform some tasks and project manager spawed some agents but then human leaves chat page and comes back, things should not be lost.
- If users stop agents working and come back and ask project manager: `what were we working on`? project-manager always looks curernt chat history + kanban tasks to provide answer. This makes system restart-safe which means agents always know where they left.

### Workflows

#### General Workflow (Must follow always)

0. Human Prompt -> 1. Project Manager (spawns or replies directly based on prompt) -> 2. Task Planner (creates project/task plan) 3. PM presents human with plan created by task planner to approve in a big nice dialog/modal with editable contents -> 4. User approves and plan gets saved in notes table 5. Task Planner creates tasks based on approved plan in kanban and reports back to PM once done 6. -> PM spawns Backend Engineer|Frontend Engineer 7. -> Code Reviewer (here if code review found issues, it will communicate accordingly and then pm will ask back to Backend Engineer|Frontend Engineer to fix it. Max of 3 turns by default but configurable in project settings) 8. -> Tester (again if tester found issues, it will communicate accordingly and then pm will ask back to Backend Engineer|Frontend Engineer to fix it. Max of 3 turns by default but configurable in project settings, same setting applies for both) -> Done (pm moves task to Done column in kanban)

	Note 1: If `Agent Plan Approval Mode` is set to `Auto-Approve`, we dont present plan to user, we just assume it is approved and proceed.
	Note 1: During the process pm call spwan other agents as well such as data engineer, performance engineer, etc based on task.	
	Note 2: We should have docs tab in activity pane which should show any notes/docs created during the conv so humans can view them any time.
	Hint: This notes and artifacts idea is similar to google's Antigravity to give you idea, you may check it out if needed on internet.

### Channels

We must support whatsapp (scan via qr code), telegram and discord. This will allowl users to communicate with pm remotely even when they are not in front of computer, their inter-communication will end up in inbox as well as channel they are communicating on. For example human might ask pm `hey, work on xyz project's abc feature and inform me when it is done`. PM will receive this and once it is done, it will reply to human on all configured channels once task is done. human can also chat generally with pm which may not be related to projects, this will be like a always available ai model for any human queries. This means pm should be smart enough to understand wheather human's message is to work on something or just a normal chat. Humans can also instruct pm to just create new project and start working on it directly while human is on vacation (unless human has already turned on auto aproval features). PM should keep human updated with all progress, kanban tasks done, wip, done, etc.

### Kanban

We will have interal kanban with columns `Backlog`, 'Working' and `Done`. Task Planner and PM will have built-in skill to move tasks in diff swimlanes/columns. Other agents cant. Humans can also move tasks around by dragging, create/edit tasks, etc.

If human moves a task from `backlog` to `working`, pm will automatically be noticed and create new conversation to work on that. Similarly, if human moves task from working back to backlog, pm stops working on that, same when human moves a task to done. You get the idea.

Tasks should be searchable, sortable, draggable, all basic standard features. Each column has diff color indicators.

It is crucial that other than description, all tasks must have checkbox `[ ]` of `Acceptance Criteria` as well as `Important Notes` agents can use for their decisions. This means agents should have special way of marking one or more of those checkboxes done as well as be able to add notes. These are separate from description.

We must also have `blocked_by` feature, this is crucial so agents can easily decide which tasks to work on first and which ones they can't until other tasks are done.

Must also have priority, date, etc other necessary stuff for tasks.

### Cron Tasks

We should have `internal` Cron Tasks system. Why? Humans can ask PM to perform certain tasks at specific times or in recurring manners. Examples of what human might say: `Create xyz project with these details on 10pm today` or `Hey, remind me to go to doctor at xyz time` or `Hey give me daily news brifing in moring at 10am` or `hey, remind me to drink water every three hours` or `hey remind me to check my emails every 5 minutes` and so on. But other than that, maybe agents can also use this internally for working on projects.

These cron tasks should be re-start safe, meaning even if app is not running and when it is re-started, it should pick it up correctly. Maybe create watcher job in system, etc. I dont know how this will work. Claude must come up with solution. Must work on Windows or any other OS. If we can achieve this internally within app, probably it will be better, not sure.
